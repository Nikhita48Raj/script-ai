# 🎬 Cinema Production Lab — Feature Guide

> A comprehensive film-making themed production intelligence platform.  
> Analyze screenplays, visualize production risk, and manage casting & storyboarding — all in one cinematic interface.

---

## 🎥 1. Script Upload & Analysis

**How to use:** Land on the homepage, drop a `.txt` screenplay file or paste script text, then click **🎬 Analyze Script**.

- **Drag & Drop**: Drop a `.txt` file directly onto the upload zone.
- **Paste Mode**: Toggle the paste editor to write or paste screenplay text manually.
- **Sample Script**: Click **Load Sample** to load a built-in demo screenplay.
- **AI Pipeline**: The backend parses your script into structured scenes, identifies characters, locations, and generates a full dependency graph plus risk analysis.

**What happens behind the scenes:**
1. Script is split into scenes based on standard screenplay formatting (`INT.`/`EXT.` headers).
2. Each scene is analyzed for characters, locations, dialogue density, and emotional tone.
3. A dependency graph is built, linking scenes by shared characters, props, and locations.
4. Cinematic metadata is heuristically generated: camera angles, shot types, lighting notes, and sound cues.

## 📋 2. Structure View

**Tab:** `📋 STRUCTURE`

A bird's-eye view of your screenplay's scene breakdown with production analytics.

### Analytics Dashboard (4 cards):
| Metric | Description |
|--------|-------------|
| **Scene Completion** | Percentage of scenes fully analyzed |
| **Shooting Efficiency** | Estimated shooting efficiency based on scene complexity |
| **Estimated Runtime** | Projected film runtime in minutes (based on word count heuristics) |
| **Expected Budget** | Heuristic budget estimate based on scene types and requirements |

### Scene Strip
- Interactive horizontal scene cards showing location, time, risk level, and emotion score.
- **Click** a scene to view its detailed metadata in the **Active Scene Slate** side panel.
- **Hover** over scenes to see cinematic metadata tooltips (shot type, camera angle, lighting, sound notes).

### Active Scene Slate
- Displays the selected scene's full details including scene type, emotion percentage, and schedule estimate.

### Script Intelligence
- Word count, page count, average dialogue percentage, peak cast count, unique locations, and unique characters.

## 🎬 3. Storyboard View

**Tab:** `🎬 STORYBOARD`

A grid of storyboard-style scene frames, designed to look like a camera viewfinder.

### Features:
- **Camera Viewfinder UI**: Each frame includes rule-of-thirds grid lines, center crosshairs, and cinematic framing overlays.
- **REC Indicators**: Flashing red recording dots with timecode (`TCR`) and ISO readouts for authentic film feel.
- **Shot Type & Camera Angle**: Displayed prominently in each frame using the heuristic metadata generated by the analysis engine.
- **Play Button Hover**: Hovering over any storyboard card reveals a play button overlay with glassmorphism effect.
- **Scene Details**: Below each frame: location, INT/EXT type, time of day, text excerpt, complexity score, and lighting notes.

---

## 🎭 4. Casting Roster

**Tab:** `🎭 CASTING`

IMDb-style character casting cards extracted from the screenplay's dialogue and scene data.

### Features:
- **Studio ID Cards**: Each character is displayed as a professional casting card with gradient backgrounds and silhouette art.
- **Role Classification**: Characters are classified as **Lead Role** or **Supporting** based on scene appearance frequency.
- **Status Badges**: Cast/Audition badges with glowing status indicators (green = cast, amber = in audition).
- **Role Match Progress**: Visual percentage bar showing heuristic "Role Match" scoring for each character.
- **Barcode Decoration**: Decorative barcode strips at the bottom of each card for studio authenticity.

---

## 🔗 5. System View (Dependency Graph)

**Tab:** `🔗 SYSTEM`

An interactive force-directed graph showing scene dependencies.

### Features:
- **Dependency Graph**: Visualizes how scenes connect through shared characters, locations, and narrative flow.
- **Narrative Mode**: Toggle between "Dependencies" and "Narrative" graph layouts.
- **Scene Filters**: Filter by location, character, or high-risk scenes only.
- **Node Interaction**: Click nodes to select scenes, hover to see connections and metadata.
- **Bottleneck Detection**: Automatically identifies the most critical bottleneck scene in the production graph.
- **Ripple Animation**: Visual ripple effect showing how delays propagate through the dependency network.

---

## 💥 6. Impact Simulation

**Tab:** `💥 IMPACT`

Run production delay simulations to see cascading effects across the entire shooting schedule.

### Features:
- **Single Scene Simulation**: Select a scene, set a delay (in days), and simulate impact across all dependent scenes.
- **Monte Carlo Analysis**: Run thousands of randomized simulations to get statistical risk metrics (P95 delay, expected cost impact).
- **Cost Impact Tracking**: See total projected cost impact based on delay cascades.
- **Affected Nodes**: Visual list of all scenes impacted by the simulated delay, with individual delay breakdowns.
- **Timeline Comparison**: Before/after timeline showing how the shooting schedule shifts with delays.
- **Risk Amplification**: Adjustable parameter to model different risk sensitivity levels.

## 🎥 7. Film Project Dashboard (Top HUD)

Always visible in the top-left corner after analysis.

- **Genre**: Heuristically determined film genre (Drama, Action/Thriller, etc.)
- **Runtime**: Estimated total runtime based on scene analysis.
- **Scene Count**: Total number of parsed scenes.
- **Risk Bar**: Color-coded gradient bar (gold → crimson) showing overall production risk level.

---

## 📜 8. Call Sheet (Center HUD)

Always visible at top-center after analysis.

- **Shoot Days**: Estimated total shooting days.
- **EXT Scenes**: Number of exterior/outdoor scenes (weather-dependent).
- **ACTION Scenes**: Count of high-action sequences.
- **Weather Risk**: Aggregate weather risk percentage for outdoor scenes.

## 💾 9. Project Management

Utility buttons available in the top-right after analysis:

| Button | Function |
|--------|----------|
| **Save** | Save the current project state to browser localStorage |
| **Load** | Restore a previously saved snapshot |
| **Export** | Download the full project data as JSON |

## 🎨 10. Cinema Theme Design System

The entire interface follows a premium cinema production aesthetic:

- **Color Palette**: Deep blacks (#0a0a0f), charcoal surfaces, cinema gold (#d4a843), crimson (#c0392b), neon blue (#38bdf8)
- **Typography**: Playfair Display (serif headings), Inter (body text), JetBrains Mono (data/code)
- **Film Grain Overlay**: Subtle SVG noise texture overlay across all views
- **Cinema Vignette**: Radial gradient darkening at screen edges
- **Projector Sweep**: Animated light sweep effect in the background
- **Film Sprocket Borders**: Decorative sprocket hole patterns on left/right edges
- **Glassmorphism Panels**: Frosted-glass panels with backdrop blur
- **Smooth Transitions**: Framer Motion page transitions (fade, scale) between all views

## 🏗️ Technical Architecture

```
Backend (FastAPI + Python)
├── app/main.py          → API endpoints & project management
├── app/models.py        → Pydantic data models (SceneCard, FilmProjectMeta, etc.)
└── app/services/
    └── script_pipeline.py → Script parsing, scene analysis, cinematic heuristics

Frontend (React + TypeScript + Vite)
├── src/AppCinematic.tsx  → Main application with all views/modes
├── src/api.ts            → API client
├── src/index.css         → Cinema design system
├── tailwind.config.js    → Custom theme configuration
└── src/components/
    ├── StoryboardGrid.tsx    → Storyboard camera viewfinder cards
    ├── CastingBoard.tsx      → IMDb-style casting cards
    ├── SceneStrip.tsx        → Interactive scene timeline
    ├── GraphCanvas.tsx       → Force-directed dependency graph
    ├── SimulationEngineUI.tsx → Delay simulation interface
    ├── InsightPanel.tsx      → AI insights & assistant
    └── ImpactTimeline.tsx    → Before/after timeline comparison
```
